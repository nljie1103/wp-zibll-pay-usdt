# wp-zibll-pay-usdt

九流网络 USDT 支付插件，为 WordPress 子比 Zibll 9.0 提供 USDT-TRC20 原生收银台支付、链上核验和订单交付。

## 付款金额提示

收银台显示的 USDT 数量是网站收款地址必须实际到账的净额。链上或交易所提币手续费由付款用户另行承担，不得从页面显示的应付金额中扣除。

## Cron 安全约定

外部 Cron 仅接受 `POST`，并且密钥只能通过 `X-Jiuliu-Cron-Token` 请求头传递。不要把 Cron 密钥放在 URL 查询参数中。

关闭“接受新的 USDT-TRC20 订单”只阻止新报价，已签发支付单仍会自动核验；只有“紧急暂停全部自动监控与结算”会暂停既有支付单。暂停、任务占用、部分失败或完全失败时，外部 Cron 不会伪报成功。

## 1.0.2 安全修复摘要

- 子比父单、子单和插件支付单使用同一 InnoDB 行锁事务，关闭订单不会被到账结算重新打开。
- 子比重新发起支付在刷新临时订单号前执行生命周期锁；涉及 USDT 时再校验登录用户或游客订单 Cookie。
- “不确定结算”由数据库原子条件保护，不能被旧请求清空、普通重放或直接拒绝隐藏。
- 人工 txid 使用已固化交易回执和完整事件分页核验，严格检查合约、事件、地址、金额、时间和唯一 txid。
- 数据库迁移、InnoDB 引擎或唯一索引不完整时，网关、轮询、Cron 和人工核验都会安全停机。

## 安全提示

插件不需要私钥、助记词或钱包密码。上线前请先备份，并使用可承受损失的小额 TRON 主网订单验收。
