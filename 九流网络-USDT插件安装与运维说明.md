# 九流网络 USDT 支付 1.0.0：安装、配置与运维说明

## 1. 本版本的适配范围

本插件是独立 WordPress 插件，不修改 Zibll 文件。它依据所提供站点中的 WordPress 7.0.1、Zibll 9.0 和实际数据库配置完成源码级适配，结算货币为 CNY。

本版可把“USDT”加入子比原生收银台，并通过 Zibll 9.0 的统一结算入口完成子比自身支持的现金网关订单，包括余额充值、会员、付费文章/资源、论坛付费、商城商品及购物车关联订单。余额、会员、内容权限、库存、销量、佣金、分成和发货仍由子比原生逻辑处理，插件不会直接篡改这些业务字段。

可执行的链上适配器只包含 **TRON 主网 USDT-TRC20**。官方合约固定为：

`TR7NHqjeKQxGTCi8q8ZY4pL8otSzgjLj6t`

## 2. 安装前准备

准备以下内容即可：

- 一个由你控制的 TRON 主网公开收款地址，以 `T` 开头；
- 建议申请一个只读 TronGrid API Key；
- 如使用自动汇率，建议申请 CoinGecko Demo API Key；
- 一个能够创建每分钟定时任务的服务器面板、宝塔、1Panel、cPanel 或系统 crontab（推荐但不是安装必需）。

插件不需要且不允许填写私钥、助记词、钱包密码、交易所密码。请先备份网站文件和数据库。

## 3. 安装步骤

1. 登录 WordPress 后台，进入“插件 → 安装插件 → 上传插件”。
2. 上传 `jiuliu-usdt-payment-1.0.0.zip`，安装并启用。
3. 左侧进入“USDT 收款 → 设置”。
4. 暂时不要启用网关，先填写收款地址、汇率和 API Key并保存。
5. 进入“系统状态”，确认 Zibll 9.0 支付接口显示正常。
6. 分别点击“测试 TronGrid”和“测试汇率”。
7. 测试通过后再勾选“启用 USDT-TRC20”。
8. 用能够承受损失的小额真实主网订单做完整验收，再正式开放。

数据库会增加两张独立表（实际前缀取决于 WordPress 配置）：

- `{prefix}_jiuliu_usdt_invoices`
- `{prefix}_jiuliu_usdt_logs`

卸载插件不会自动删除财务核对记录。

## 4. 推荐设置

### 基础收款

- 收款地址：只填 TRON 主网公开地址；插件会执行 Base58Check 校验。
- TronGrid API Key：生产环境建议填写，可提高查询额度；它只能用于读链，不具备转账权。
- 最低订单金额：默认 ¥1。过低金额可能无法生成足够安全的唯一六位小数尾数。
- 最高订单金额：根据站点风险偏好设置；大额订单建议人工复核。

### 订单与异常

- 备用有效期：默认 15 分钟。正常情况下插件使用子比原生支付单截止时间，此值仅作兼容兜底。
- 过期到账观察期：默认 24 小时。
- 子比订单关闭后继续观察到账：建议开启。
- 允许前台提交交易哈希：建议开启，便于处理交易所截断小数或用户误付。
- 管理员邮件：建议开启并确认 WordPress 发信正常。

### 汇率

- 固定模式：最可控。填写“1 USDT 对应多少 CNY”。
- 自动模式：从 CoinGecko 读取 USDT/CNY，支付单创建时锁定汇率，15 分钟缓存。
- 备用固定汇率：自动接口失败、超时、数据过期或偏差超限时使用。
- 偏差熔断：默认 10%。自动值相对备用固定值偏差过大时拒绝自动值。
- 汇率加成：正数会增加用户应付 USDT，可覆盖波动或提现成本；负数表示优惠。

CoinGecko 是独立市场数据聚合服务，并非 Tether 官方人民币结算价，也不是任何交易所的成交承诺。因此自动模式可以一键使用，但上线前必须把备用固定汇率设置到合理值并执行后台测试。追求确定金额时使用固定模式。

## 5. 监控是自动的，监控哪个网址

插件有三条互补路径：

1. 支付弹窗打开时，子比前端自动轮询支付状态；
2. 插件每分钟注册一次 WP-Cron 任务；
3. 推荐由服务器每分钟调用插件自己的受保护接口。

本站监控入口为：

`https://blog.jiuliu.org/wp-json/jiuliu-usdt/v1/cron`

这个 URL 不是某笔交易的网址。收到合法 Cron 请求后，插件会使用 TronGrid 的只读账户历史接口查询配置的收款地址，并核对当前待处理支付单。

后台“USDT 收款 → 系统状态”会生成带当前随机密钥的完整命令。Linux/宝塔/1Panel 推荐：

```cron
* * * * * curl -fsS -X POST -H "X-Jiuliu-Cron-Token: 后台显示的随机密钥" "https://blog.jiuliu.org/wp-json/jiuliu-usdt/v1/cron" >/dev/null 2>&1
```

如果面板只支持访问 URL，可使用系统状态中折叠显示的兼容 URL，但请求头方式更安全，因为密钥不容易进入代理访问日志。

不添加服务器任务时插件仍会工作，但 WP-Cron 只在网站出现访问请求时触发，低访问量站点可能延迟。正式收款建议添加服务器任务。外部 Cron、WP-Cron 和浏览器轮询即使重叠，也有互斥、原子认领和幂等保护，不会重复发货。

## 6. 子比订单关闭与 USDT 监控

插件监听 Zibll 9.0 原生 `order_closed` 事件，并通过稳定的 `payment_id` 同步插件支付单，而不是依赖每次发起支付都会变化的临时订单号。

| 场景 | 插件行为 | 是否自动发货 |
|---|---|---:|
| 正常有效期内确认到账，子比主单和全部子单均待支付 | 调用子比统一结算入口 | 是 |
| 子比订单已关闭，关闭后监控开关开启 | 观察期内继续查询；到账转人工 | 否 |
| 子比订单已关闭，关闭后监控开关关闭 | 停止该关闭单的自动查询；可后台手工核验 txid | 否 |
| 用户先广播交易、随后手动关闭，之后交易才确认 | 重新读取主单和全部子单；到账转人工 | 否 |
| 超时未支付 | 定时任务调用子比原生关闭逻辑并恢复主题/商城库存 | 否 |
| 少付、多付、过期到账或切换了其他支付方式 | 绑定有效链上证据并转人工 | 否 |

“关闭后停止监控”节省的是 TronGrid API 查询额度和服务器处理量，不是 TRON Energy 或 TRX。插件只读公开链上数据，不签名、不广播交易，因此监控本身没有链上手续费。

## 7. 链上核验规则

自动结算必须同时通过：

- 收款地址与该支付单锁定的地址一致；
- 网络为 TRON 主网；
- 合约严格等于官方 USDT-TRC20 合约；
- TronGrid 返回 `only_confirmed=true` 的已确认记录；
- 代币精度严格为 6；
- 类型为转账，且方向是转入；
- 原始整数金额与支付页六位小数完全相等；
- 区块时间位于该订单有效期内；
- 交易哈希为 64 位十六进制，且全站只能认领一次；
- 子比主支付单、金额、支付方式和全部关联子订单仍允许结算。

公开提交 txid 时，金额或时间不属于当前支付单的交易不会先占用全局 txid；这可防止用户拿他人的公开交易哈希抢占订单。管理员勾选“强制补单”也只放宽金额/时间条件，已关闭、已更换方式或状态混乱的子比订单仍被安全拦截。

## 8. 其他链能否增加

可以，但 1.0.0 不会识别其他网络。Tether 当前官方支持多种协议，包括 Tron、Ethereum、Avalanche、Solana、TON、Aptos 等；每条链的官方资产、确认语义、RPC/API、地址格式和交易解析方式不同。

- Solana-USDT：适合作为下一条链，用户手续费低；需要单独实现 Solana RPC、SPL Token 指令解析、finalized 确认、Mint 校验和分页游标。
- Ethereum/Avalanche 等 EVM 链：可以抽象成 EVM 适配器，但仍需逐链固定 chain ID、官方合约、RPC、确认数和日志分页。
- TON/Aptos：数据模型不同，需要独立适配器。

新增一条链属于中等工作量，主要改造链上核验层和前台网络选择；已经完成的子比结算、订单关闭、后台订单和幂等层可以复用，不必重写整个插件。

## 9. 正式上线验收清单

请在测试账号上依次完成：

1. 小额余额充值；
2. 会员购买；
3. 付费文章或下载资源；
4. 商城自动发货商品；
5. 购物车多商品/关联订单；
6. 重复提交同一 txid，第二次必须失败；
7. 少付、多付和晚于截止时间的转账，必须不发货；
8. 支付方式切换后给旧 USDT 金额转账，必须转人工；
9. 用户关闭订单后转账，必须转人工且库存不得重复扣减；
10. 关闭“关闭订单到账观察”后创建并关闭新订单，后台自动扫描不应继续监控该单；
11. 关闭付款页面后等待服务器 Cron，正常付款仍应自动完成；
12. 检查邮件、后台日志和子比订单详情中的链上凭证。

## 10. 运维与故障处理

- 一直等待：先到“系统状态”测试 TronGrid，再检查服务器 Cron 和 WordPress 时区。
- HTTP 429：配置 TronGrid API Key，确认 Cron 没有被错误设置成高于每分钟一次。
- 自动汇率回退：检查 CoinGecko Key、服务器出站 HTTPS、备用汇率和偏差阈值。
- 进入人工处理：先在 Tronscan 核对 txid，再核对子比主单/子单、库存和权益；不要在未核对前重复发货。
- 退款：只能在外部钱包人工完成，并把退款 txid 记录到财务备注。插件没有私钥，因此不会自动退款。
- 更新主题：先在测试环境复测上述验收清单，尤其是支付成功和订单关闭事件。

## 11. 已完成的审查与限制

本包已执行 PHP 7.1 语法检查、JavaScript 语法检查、静态安全审查、子比 9.0 源码接口比对、并发/幂等回归测试，并对 TronGrid 与 CoinGecko 的公开接口格式做了实网读取验证。

这些检查不能替代你服务器上的小额真实支付验收：最终行为还会受到主机网络、MySQL、缓存插件、防火墙、邮件配置、第三方 API 限额及未来 Zibll 更新影响。第一次上线请保持小额并观察后台日志。
